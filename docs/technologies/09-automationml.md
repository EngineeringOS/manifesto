# AutomationML

AutomationML is one of the most important industrial exchange references in the draft. It matters because it gives EngineeringOS a serious standards-facing path into manufacturing and industrial automation ecosystems.

## How EngineeringOS Uses It

EngineeringOS should study AutomationML deeply and treat it as an importer and exporter boundary for industrial semantic exchange. It is especially relevant where compiled engineering meaning needs to reach plant, control, or integration workflows.

## Boundary

AutomationML is a standards boundary, not the native internal model. EngineeringOS should map to it through explicit compiler and plugin contracts rather than letting it define the internal ontology wholesale.
